Home Page

Technical Description

The header was enclosed in a <header> tag for readability. The navbar was made by having a "display: flex" div with a ul that has hidden bullet points. The position of the header was set to fixed to ensure it doesn't move with the content, and the z-index was set to 50 to allow others to use a z-index between 1 and 49 for other purposes. The navbar links have a hidden underline text-decoration that is revealed when hovered over, using the "transition" property to set a duration. If the current page is in the header, that link is given the "currentlink" class to keep it underlined constantly for clarity.

Classes were used to ensure styling wasn't repeated. Under the <main> tag, a "topsect" class was used to define the heading div, with the "background-size" and "background-repeat" properties used to position the background image properly. Unlike the rest of the content on the page, this div spans the full length to give it further importance. The Headings on this div have inline css to position then in a well-spaced manner. After the topsect div, "wrapper" class div is opened to define margins and padding for the main content of the site. A "mission" class div is used similarly to the "topsect" div.

Flipping cards were used to showcase the Content Pages. The "postrow" class div is set to "display: flex" and "flex-wrap: wrap" to allow flexibility in spacing. The individual posts are defined with "postfront" and "postback" divs to write the front and back of the cards. On hover, the "transform: rotateY(180deg)" css style is used to flip from the front to the back. The entire "post" div enclosed in an <a> tag to allow it to link to the content pages.

The footer includes key links to other pages as well as two images of the class "footerlogo" on the left and right edges. An unordered list is used to display the content pages, with each column of links enclosed in a "footertext" class div. The "::marker" pseudo-element was used to change the color of the bullet points in the footer.

A Favicon was added using the link tag with the attribute image/x-icon. To allow for responsiveness, some elements such as headings have their sizes defined in "vw" units, which is 1% of the width of the viewport.

Custom fonts were included in the files, and were defined using the "@font-face" css tag. Gradients were used in backgrounds and the header to prevent a static look, but also include a "background" property set to a static color underneath to account for browsers that do not support gradients.

Media Queries were used to add responsiveness to the site, resizing a few key font-sizes, padding values, and div sizes.

Accessibility

Contrasting Colors were used between background and foreground elements to ensure readability. Alt tags were included for all images. Structural elements such as <header> and <main> were used to allow assistive technologies to better understand the structure of the page. The language of the document was defined in the <html> tag as well. Aside from the header, all links were given an underline to indicate that they are clickable links. The header links gain an underline when hovered over as well.

Table Page

Technical Description

Similar to the home page, a "topsect" div is used for the heading, and the "wrapper" div contains the main content of the page. The table details the sub-targets set by the UN for SDG 14.

All Heading Row cells include a tooltip that is shown when hovered over.This is done by using <span> with the "tooltipcontent" class to detail the tooltip contents, and the cell contents with a "tooltipdiv" class. The opacity of the tooltip div is set to 0 by default, and set to 1 when hovered over. The z-index of the tooltip is set to 1 to ensure it is shown over the standard webpage content. The "position: relative" tag as well as "bottom: 105%" and "left: 50%" ensures it is centered and slightly above the cell.

To implement zebra striping, each table row has a class of either "tablerowA" or "tablerowB", which defines the coloring of the row.

Thumbnail images are included in the first column of every row. All images used in the table grow slightly when hovered over using the "transform: scale()" property. This was used instead of the height property to prevent other content of the table moving when the image grows.

The images in the last column of each row link to an external site where the user can learn more about each target.

Accessibility

Contrasting Colors were used between background and foreground elements to ensure readability. Alt tags were included for all images. Structural elements such as <header> and <main> were used to allow assistive technologies to better understand the structure of the page. The language of the document was defined in the <html> tag as well. All images that are hyperlinked grow slightly on hover to indicate that it is clickable. Box-shadows are used to soften edges and increase contrast.

Content Page

Technical Description

A return to top button was implemented, positioning it using "position: fixed", "bottom: 0" and "right: 0". The cursor becomes a pointer when hovering over the image to indicate that it is a clickable button. While internal linking could've been used, JavaScript was used instead to implement a smooth scroll, rather than a jump to the top. Further, JS was used to hide the button when the page was near the top, using the "document.body.scrollTop" value to define whether the element was given the "visible" class. This script in linked externally from script.js.

A contents pane was implemented with internal links for easy navigation within the page. Nested lists were used to structure headings and sub-headings, with the "subol" class giving sublists the "list-style-type: lower-roman" property and a reduced line height.

A class called "sidebyside" was defined to give divs the "display: flex" and "justify-content: space-between" properties, to allow for images next to sections of text. Each major section is seperated with a <hr> tag.

Accessibility

Contrasting Colors were used between background and foreground elements to ensure readability. Alt tags were included for all images. Structural elements such as <header>, <main> <h3> (For main article headings) and <h4> (For article sub-headings) were used to allow assistive technologies to better understand the structure of the page. Each heading tag has an id to allow for internal linking. The language of the document was defined in the <html> tag as well. Box-shadows are used to soften edges and increase contrast for images. In order to give images a caption, the <img> tag is defined within a <figure> tag, with a <figcaption> tag following it for the caption.

Challenges and Lessons Learned

  • A major challenge faced was learning the process of positioning elements in the desired way. The use of divs to group elements, various position CSS properties, padding, and margins was learned.

  • Keeping the markup of the website readable was challenging until proper indentation and frequent commenting was used.

  • Understanding when to use external, internal, and inline CSS, as well as ids instead of classes, was crucial since the project was implemented with group members.

  • When using internal links to jump to sections of the page, the start of the content would be covered by the header. To remedy this, the scroll-padding-top property was applied to the html tag to offset the height of the header.

  • Header links turning purple when visited was not cohesive to the design, so the ":visited" and ":link" color was kept the same. However for footer links ":visited" links turn a light grey. Standard link colors were preserved for other links.
